# 002 - Utilitarian Dense Design

## 설계 철학

**효율성과 정보 밀집도** 최우선으로 설계되었습니다. 실무 전문가와 개발자를 위한 "도구"로서, 최대한 많은 정보를 한 화면에 표시하고 빠른 작업 흐름을 지원합니다.

### 핵심 원칙
- **정보 밀집도 최대화**: 모든 중요 정보가 한눈에 들어옴
- **최소한의 장식**: 기능 외 모든 요소 제거
- **다크 모드 기본**: 장시간 사용 친화적
- **키보드 친화적**: 마우스 의존도 최소화

## 주요 선택 사항

1. **레이아웃**: 2컬럼 고정 (달력 + 태스크 패널, 우측 스크롤)
2. **색상 팔레트**:
   - 배경: #1a1a1a (깊은 검정)
   - 헤더/카드: #0d0d0d (더 어두운 검정)
   - 액센트: #4ade80 (밝은 초록)
   - 텍스트: #e0e0e0 (밝은 회색)
3. **폰트**: 시스템 산스 (-apple-system, Roboto, Segoe UI)
4. **간격**: 최소화 (0.4rem~0.8rem)
5. **셀 표시**: 1px 그리드 분리선으로 엄격한 구조

## 강점

✅ **빠른 작업 흐름**: 한 화면에서 모든 작업 관리  
✅ **높은 정보 밀집도**: 일주일 또는 한 달의 전체 개요 확인 가능  
✅ **기술 친화적**: 개발자, 설계자 등 도구 사용에 익숙한 사용자 선호  
✅ **다크 모드 기본**: 장시간 사용에 피로 감소  
✅ **확장성 우수**: 추가 기능(필터, 검색, 통계)을 쉽게 추가 가능

## 약점

❌ **가독성 저하**: 작은 텍스트와 복잡한 레이아웃  
❌ **학습 곡선**: 새 사용자에게 인터페이스 이해가 어려움  
❌ **시각적 피로**: 높은 밀집도로 인한 눈 피로  
❌ **모바일 부적합**: 컴팩트 디자인이 작은 화면에 맞지 않음  
❌ **초보자 진입장벽**: 단순히 일정을 관리하려는 사용자에게는 과도함

## 최적 사용 시나리오

- ⚙️ **개발자/기술자** (CLI, 개발 도구에 익숙한 사용자)
- 📊 **대량 데이터 관리** (수십 개 이상의 태스크)
- ⚡ **빠른 입력/관리** (마우스보다 키보드 입력 선호)
- 🔄 **다중 프로젝트 추적** (여러 뷰를 한눈에 봐야 하는 경우)
- 💻 **전문 업무 환경** (엄격한 일정 관리가 필수)

## 기술 스택

- **스타일링**: 인라인 CSS (깔끔한 클래스 기반 구조)
- **폰트**: 시스템 산스 폰트
- **상호작용**: Vanilla JavaScript (빠른 응답성)
- **호환성**: 모든 최신 브라우저, CLI 도구 스타일

## 확장 가능성

이 디자인을 기반으로 다음을 추가할 수 있습니다:
- 커스텀 우선순위 색상 코딩
- 상태별 필터링 (완료, 진행중, 미시작)
- 키보드 단축키 지원
- 데이터 내보내기 (CSV, JSON)
- 실시간 통계 대시보드
- 팀 협업 기능

## UI 가이드라인

- **선택의 피로 최소화**: 기본값 제공
- **시각적 피드백**: 호버 상태와 선택 상태 명확
- **일관된 간격**: 모든 요소가 4px 그리드 기반
- **색상 의미**: 우선순위별 일관된 색상 코드
